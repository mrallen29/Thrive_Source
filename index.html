5. Explain, in a couple of paragraphs, what you think are two major challenges around DNS configuration for less-technical customers hosting websites.


Answer: One big challenge with DNS configuration is understanding the different settings. Things like A records, CNAMEs, and MX records can be confusing, and changing the wrong one can make a website or email stop working.

Another challenge is that DNS changes don’t update instantly. It can take minutes or even hours to fully work, which can be frustrating. If something goes wrong, it’s hard for non-technical users to know if they made a mistake or if they just need to wait.

7. How would you set up an http 301 status redirect from “/netlify/anything” on your site, to https://www.google.com/search?q=anything. Please provide the redirect formatting here. Now, how about a proxy redirect? Please add that proxy redirect rule directly to your site. (Please add the proxy redirect to your website!)


Answer: To set up an HTTP 301 redirect from /netlify/anything to

https://www.google.com/search?q=anything



I added the following rule in a _redirects file:

/netlify/:query https://www.google.com/search?q=:query 301!



his rule captures anything after /netlify/ and redirects it permanently (301) to Google Search with the same query parameter.




For a proxy redirect, I used this rule:


/netlify/:query https://www.google.com/search?q=:query 200



This makes it appear as if the content is served from my website, but it actually fetches the result from Google. I placed the _redirects file in the root directory of my project alongside index.html and styles.css. After deployment, Netlify automatically processes these rules to handle the redirects.
